# 详情信息与异常态组件规则V1.1

这份文档主要回答两类问题：

- 详情信息为什么应该优先用统一的描述组件、时间线和查看态抽屉
- 系统异常页和页内异常态为什么不能每次都各写一套

如果你要补的是以下能力，优先看本文档：

- Drawer
- DescriptionList / DetailInfo
- Timeline
- ExceptionState
- Upload / Attachment
- DateRangePicker

## 1. 详情信息组件

### 1.1 DescriptionList / DetailInfo

适用场景：

- 详情页基础信息
- 详情页财务信息
- 主数据只读信息区
- 查看态说明卡片

固定规则：

- 优先复用统一的描述列表组件
- 默认使用2列、3列或4列栅格，不再在每个页面手写`InfoGrid`
- 标签和值采用稳定的纵向排列
- 长文本允许换行，但整体栅格节奏保持稳定

不建议：

- 在每个详情页里重新写一套`description-grid`
- 用disabled表单控件假装详情展示

### 1.2 AttachmentPanel

第一版能力边界：

- 上传触发
- 附件列表
- 下载
- 删除
- 失败重试

固定规则：

- 编辑态和详情态复用同一个组件
- 详情态默认只展示和下载，不展示上传、删除
- 附件区放在“其他信息”或“附件与备注”卡片，不混进基础字段区

不建议：

- 只用一个文本字段冒充附件
- 每个业务页各自发明附件列表样式

### 1.3 Timeline

适用场景：

- 操作日志
- 变更轨迹
- 系统处理轨迹

固定规则：

- 第一版优先做通用垂直时间线
- 审批记录表格可暂时保留，审批流图后续再独立补
- 时间线优先强调时间、动作和结果，不做装饰性流程图

## 2. Drawer

Drawer属于查看态优先组件。

固定规则：

- 默认采用`createPortal + fixed inset-0`
- 遮罩覆盖整个视口
- 从右侧覆盖主体内容
- 锁定body滚动
- 头部保持固定高度和关闭按钮口径

优先替代的场景：

- 消息详情
- 导出任务详情
- 轻量核对型查看页

不建议：

- 用页面内部绝对定位伪装抽屉
- 用分栏硬挤列表宽度代替抽屉

## 3. DateRangePicker

第一版不依赖第三方库。

固定规则：

- 触发器 + 轻量弹层 + 原生`date`输入
- 默认支持`今天 / 近7天 / 近30天`
- 支持清空和确认
- 优先用于系统级工具条和查询页顶部

不建议：

- 用假按钮文案长期占位
- 每个页面各写一套日期范围弹层

## 4. ExceptionState

第一版固定支持：

- 403
- 404
- 登录过期 / 会话失效
- 系统维护 / 功能未开放

固定规则：

- 同一组件承接系统状态页与页内异常态
- 必须有明确主动作，例如返回首页、重新登录、联系管理员、稍后重试
- 系统状态页和页内异常卡片要共享同一语气和视觉边界

## 5. 落地优先级

这类组件如果已经在工程里重复出现，就按以下顺序收敛：

1. 先抽共享组件
2. 再替换现有代表页
3. 再补进Design System
4. 最后写进案例和模板

如果没有进入规则文档、Design System和案例目录，它就还不算真正沉淀完成。
