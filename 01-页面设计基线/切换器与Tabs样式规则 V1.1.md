# 切换器与Tabs样式规则V1.1

这份文档不讨论颜色值，也不讨论具体业务字段。

它主要回答的是这些问题：

- 为什么不能把所有“切换”都做成同一种样式
- 什么叫导航型Tab，什么叫模式切换，什么叫筛选Chip
- 什么时候该用线性Tab，什么时候该用分段控件，什么时候该用轻量Chip
- 为什么有些圆角应该收，有些可以更圆一些

如果只记住一句话：

- **同一语义统一样式，不同语义允许不同样式**

也就是说，真正应该统一的不是“所有地方都长一样”，而是：

- 工作台Tab内部统一
- 状态分组Tab内部统一
- 详情内容Tab内部统一
- 分段控件内部统一
- 筛选Chip内部统一

## 1. 先分清5类切换

### 1.1 工作台Tab

典型位置：

- 顶部工作台页签
- 首页Tab、消息中心Tab、业务页签

本质：

- 多任务工作区切换

推荐样式：

- 线性Tab
- 小字号
- 低装饰
- 激活态以下划线或边线强调
- 默认48px高，保持紧凑但不压迫
- 文字、icon和关闭按钮垂直居中；关闭按钮保留稳定点击热区

不建议：

- 做成大圆角胶囊
- 做成筛选Chip风格
- 激活态铺大面积背景色块

### 1.2 状态Tab

典型位置：

- 列表页中的`全部 / 待审核 / 已完成 / 已取消`
- 按状态快速切换数据分组的区域

本质：

- 数据分组导航

推荐样式：

- 线性Tab优先
- 激活态可用主色文字 + 下划线
- 整行保留满宽细分割线
- 可显示数量

不建议：

- 做成大胶囊按钮式切换
- 和查询区的筛选按钮混成一排

### 1.3 内容Tab

典型位置：

- 详情页里的`基础信息 / 日志 / 审批记录 / 变更轨迹`
- 多块信息内容的平级切换

本质：

- 内容区域切换

推荐样式：

- 线性Tab
- 层级比状态Tab更弱一点
- 不抢主操作区

不建议：

- 做成强对比胶囊
- 做成像主按钮一样的高强调切换

### 1.4 分段控件

典型位置：

- `按人员添加 / 按条件添加`
- `紧凑 / 中等 / 宽松`
- `列表视图 / 卡片视图`
- `处理中 / 导出成功 / 导出失败 / 已下载`这类更偏“模式/过滤”的切换

本质：

- 操作模式或视图模式切换
- 较轻的数据过滤切换

推荐样式：

- 胶囊分段控件可以成立
- 激活项可以是胶囊块，但整体仍应保持轻量
- 可用浅背景激活，不一定要下划线
- 默认与常规按钮、输入控件保持同一高度基线，优先32px

适用前提：

- 这组切换更像“控制台开关”而不是页面导航
- 切换频率高，希望用户快速扫一眼知道当前模式

### 1.5 筛选Chip

典型位置：

- 快速过滤项
- 多选标签筛选
- 轻量条件集合

本质：

- 筛选器

推荐样式：

- 小体积Chip
- 轻圆角
- 可多选或可批量取消

不建议：

- 用Chip冒充Tab
- 用Chip承载页面主导航

## 2. 一个简单判断法

先问这次切换到底在干嘛：

- **在切页面或切工作区**：用工作台Tab
- **在切数据分组**：优先状态Tab
- **在切内容区域**：用内容Tab
- **在切视图/模式/轻过滤**：用分段控件
- **在做快速筛选或多选条件**：用筛选Chip

如果一个切换既不像导航，也不像内容分区，而更像“控制台状态过滤”，那就不要硬套线性Tab。

## 3. 圆角怎么控制

### 3.1 线性Tab

推荐：

- 不依赖圆角做主视觉
- hover可以有很轻的背景
- 激活态核心是文字、下划线、边线

结论：

- 线性Tab不应做成明显的胶囊圆角块

### 3.2 分段控件

推荐：

- 可以比线性Tab更圆，也可以是胶囊激活态
- 但圆润感只属于分段控件，不要扩散到工作台Tab和状态Tab
- 激活项可以有浅背景胶囊块，但不应再额外包一层厚重灰底边框容器
- 分段控件所在工具条如果本身已经有边线或卡片边界，外层容器优先透明，避免再套一层“输入框式”壳
- 如果位于系统页或列表页工具条中，控件本身优先保持32px高，整行工具条优先控制在48px左右

建议口径：

- 默认允许胶囊激活项
- 外层容器优先透明或极弱存在感，不做“输入框式”灰底边框外壳
- 不做高饱和、厚重、过强拟物的胶囊块

### 3.3 筛选Chip

推荐：

- 可以比Tab更圆一点
- 但仍然要克制

原因：

- Chip本来就不是主导航，更偏辅助过滤

## 4. 这套项目里的建议搭配

### 应继续保持线性Tab的

- 顶部工作台页签
- 采购订单状态Tab
- 详情页内容Tab

### 更适合分段控件的

- 导出任务中心状态切换
- 消息预览面板中的`消息 / 通知 / 待办`
- 行高切换
- 视图模式切换
- `按人员添加 / 按条件添加`

### 更适合筛选Chip的

- 快速过滤条件
- 多选标签过滤
- 辅助筛选集合

## 5. 常见误区

### 误区1：所有切换统一成一种样式

问题：

- 语义混乱
- 页面层级不清
- 用户难以判断当前是在“导航”还是在“筛选”

### 误区2：为了显眼，把所有Tab都做成浅底胶囊

问题：

- 后台会显得太软、太圆、太像C端
- 视觉噪音上升

补充：

- 这个结论主要针对工作台Tab、状态Tab、内容Tab
- 分段控件本身允许胶囊感，但不应再叠厚重外壳

### 误区4：为了“精致”把分段控件所在工具条做得过高

问题：

- 垂直节奏发松
- 和按钮、搜索框、checkbox不在同一视觉基线
- 系统页会显得发宽、不够利落

### 误区3：把状态Tab和筛选按钮混用

问题：

- 用户不知道自己是在切分组还是在改筛选条件

## 6. 规范落地方式

后续如果遇到新的切换器，先不要直接写样式。

推荐顺序：

1. 先判断语义归属：工作台Tab / 状态Tab / 内容Tab / 分段控件 / 筛选Chip
2. 再决定它应该继承哪套视觉规则
3. 最后才落到代码和Token

如果一个新切换器无法归类，优先回看本文档，而不是先发明第6种样式。
